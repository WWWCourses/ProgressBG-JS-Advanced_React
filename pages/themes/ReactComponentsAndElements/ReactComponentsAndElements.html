What is a React Element

What is a React Element

Overview


  		"use strict";

  		var element = React.createElement("h1", null, "Hello, world");
Elements are the smallest building blocks of React apps
An element describes what you want to see on the screen
React element is an object, representing a DOM node
React elements are plain objects, which are cheap to create and update (unlike DOM Elements)
ReactDOM takes care of updating the DOM to match the React elements

Creating React Elements

To create a react element we can use either:

React.createElement() method

  		const h1 = React.createElement("h1", {className: 'green'}, "Hello, world!");
  		ReactDOM.render(
  		    h1, document.getElementById('root')
  		);
JSX syntax (the preferred way)

  		const h1 = 
Hello, world!

  		ReactDOM.render(
  		    h1, document.getElementById('root')
  		);

React Element object

React Elements, created by React.createElement() or JSX, creates an object like:

      Object
        type: 'h1',
        key: null,

        props: {
          className: 'green',
          children: 'Hello, world!'
        }

Rendering an Element into the DOM

To render a React element into the DOM, we use the ReactDOM.render() method

  		ReactDOM.render(
  		    h1, document.getElementById('root')

What is a React Component?

What is a React Component?

Components are the building blocks of any React app.
Thinking in components allows us to split the UI into independent, reusable pieces, and work with each piece in isolation.
A React Component is a JavaScript class or function that optionally accepts inputs i.e. properties(props) and returns a React Element describing what should appear on the screen.
React lets you define components as classes or functions.

Component inheritance?

All the internal React Component logic comes from the React Component class
However, it isn’t recommended to use the concept of inheritance for your custom class components more than that.
In React code reuse is primarily achieved through composition rather than inheritance

Rendering a Component

Rendering a Component

React elements can represent DOM elements or user-defined components

      // React element, representing a DOM element:
      const element = <div />;

      // React element, representing a user-defined component:
      const element = <HelloWorld />;

Naming conventions

User-Defined Components Must Be Capitalized
React treats components starting with lowercase letters (like '<div />') as DOM elements.
And components starting with capital letter as user-defined, which have to be declared in the scope
Reference: React treats components starting with lowercase letters as DOM tags
